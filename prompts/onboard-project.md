# Codex Project Onboarding Task

Read `AGENTS.md`, `standards/development-workflow.md`, `standards/gamemaker-projects.md`, and `standards/project-onboarding.md` before inspecting the project.

## Goal

Determine whether this is a new blank project or an existing project and populate `docs/00-project-inventory.md` without making gameplay changes.

## Required work

1. Review Git status.
2. Locate all `.yyp` files and apply the zero, one, or multiple project rule.
3. Inspect the project structure, resource format, naming, folders, rooms, ownership, persistence, data, extensions, and protected systems.
4. Record verified facts in the inventory.
5. Mark unresolved facts as `Unknown` and list questions requiring human input.
6. Report inconsistencies and current build information without guessing.

## Constraints

- Do not create or modify gameplay resources.
- Do not normalize existing architecture.
- Do not fabricate metadata or project decisions.
- Do not compile unless a configured command is available and execution is authorized.

Provide the completion report required by `AGENTS.md` and stop before commit approval.
