# Reviews

Store durable milestone and feature reviews here when they will help future work understand what was verified and decided.

Use ordered, descriptive names such as:

```text
0001-project-foundation.md
0002-player-movement.md
0003-camera.md
```

Each review should record:

- Intended behavior
- Changed resources
- Test results
- Findings
- Decisions
- Follow-up tasks
- Commit reference

Do not store transient build output or duplicate the full implementation task. Reviews should preserve the decisions and evidence needed for later maintenance.
